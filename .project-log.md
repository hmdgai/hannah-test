## Session Notes

### 2026-04-30

- The Vite override (`"overrides": { "vite": "^7" }`) was added defensively — current resolution is 7.3.2, no Vite 8 in the tree, but the override prevents a future `npm install` from upgrading.
- Confirmed `BaseLayout.astro` cannot reference `import.meta.env.PUBLIC_SITE_URL` directly — Vite's static env replacement combined with const declarations there triggers misleading "Unterminated string literal" errors. Use `Astro.site` with `Astro.url.origin` fallback instead. (Documented in `.claude/skills/contactform.md`.)
- The Cloudflare adapter ASSETS-binding patch is wired via `npm postinstall` and is idempotent. Re-running reports `0 replacements, 5 already-done`.
- Astro 6.1 release brought no breaking changes. Sharp codec defaults, advanced SmartyPants, i18n fallback routes — none warranted code changes for clinic sites.
