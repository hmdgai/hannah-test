### 2026-05-20 (later — Sanity dev-mode compatibility fixes)

**Status:** completed (uncommitted)
**Task summary:** Fixed two crash modes when running `npm run dev` with the Sanity integration + Cloudflare adapter (platformProxy enabled). (1) Added `compatibility_flags = ["nodejs_compat"]` to `wrangler.toml` — without it, Sanity's CJS-emitting React deps trip "module is not defined" in the Workers runtime. (2) Added `vite.optimizeDeps.exclude` for `sanity`, `@sanity/vision`, `@sanity/astro`, plus `ssr.noExternal: ['@sanity/astro']` to `astro.config.mjs` — Vite's dep optimizer was producing missing-chunk errors against Studio's deep dependency graph. After both, `astro dev` boots cleanly in ~32s and `/admin` returns HTTP 200.
**Files changed:**
- `wrangler.toml` — added `nodejs_compat` flag
- `astro.config.mjs` — added `vite.optimizeDeps.exclude` + `vite.ssr.noExternal` for Sanity
**Notes:** Tested locally — homepage and /admin both respond. `nodejs_compat` also applies to production Workers; adds Node.js polyfills but no measurable perf cost for this site. Future client clones inherit this config automatically.
**Next action:** none
