# Fallback Redirect — "Rank Math → Redirect to Homepage" Equivalent

Reusable pattern for HMDG Astro + Cloudflare Pages clinic sites.

Reproduces the Rank Math WordPress feature where any unmatched URL is
301-redirected to the homepage instead of rendering a 404 page. Useful
during WordPress → Astro migrations as a safety net for old links you
haven't explicitly mapped.

---

## When to use

- Migrating a site from WordPress to Astro on Cloudflare Pages.
- You want any leftover old URLs (random crawler hits, forgotten
  backlinks, deep WP permalinks) to land on `/` instead of 404.
- You've already mapped your **high-value old URLs** to **specific
  new URLs** (do this first — don't skip it; see SEO note below).

## When NOT to use

- Building a fresh site with no old URLs to preserve. Real 404s are
  the correct response for genuine "this page doesn't exist."
- A site where 404s are expected (e.g. user-generated profile URLs).

---

## ⚠️ SEO note (read before deploying)

Redirecting all 404s to `/` is what Rank Math calls "Fallback Behaviour"
but Google considers it a **soft 404 / soft redirect anti-pattern** if
overused.

**Best practice migration order:**

1. Export top old URLs from Search Console (last 12 months).
2. Map each one to a **specific** new URL → add to
   `astro.config.mjs` `redirects:` or as specific lines in
   `public/_redirects`.
3. ONLY THEN enable this catch-all fallback for the long tail.

The fallback wins back link equity for random/unmapped URLs but should
not be a substitute for proper specific redirects on the URLs that
matter.

---

## Architecture (correct as of Astro 6 + @astrojs/cloudflare)

This pattern uses **Astro middleware**, NOT `public/_redirects`. The
distinction is critical:

```
┌──────────────────────────────────────────────────────────────────┐
│                      Cloudflare Pages Edge                        │
│                                                                   │
│  Step 1: Static asset check                                       │
│           ├─ /about/         → /about/index.html exists  → 200    │
│           ├─ /images/x.webp  → file exists               → 200    │
│           └─ no match → fall through ↓                            │
│                                                                   │
│  Step 2: public/_redirects evaluation                             │
│           ├─ Specific old → new mappings (preserve link equity)   │
│           └─ no match → fall through ↓                            │
│                                                                   │
│  Step 3: Astro Worker (generated by @astrojs/cloudflare)          │
│           ├─ Detects 404 (no static, no SSR route)                │
│           ├─ Calls app.render() to produce 404 response           │
│           └─ MIDDLEWARE intercepts → returns 301 to /             │
└──────────────────────────────────────────────────────────────────┘
```

### Why NOT use `public/_redirects` with a wildcard

A wildcard like `/* / 301` in `_redirects` BREAKS THE BUILD on
`@astrojs/cloudflare`:

- The build's prerender phase spins up a Wrangler simulator.
- The simulator processes `_redirects` against requests like `GET /`
  during the page-rendering loop.
- The wildcard catches `/` itself, returns 301, the simulator follows
  the redirect, infinite loop → "redirect count exceeded" build error.

Even with explicit homepage exclusions (`/  /  200`), the build fails
because the static `index.html` doesn't exist yet at the time of
prerender — the simulator can't find a static file to serve.

`_redirects` works for **specific** known-old → known-new redirects,
but not for a true catch-all wildcard.

### Why middleware works

Astro middleware runs **inside the Worker**, AFTER Cloudflare's
static-file lookup has failed. By that point the build is finished
and only real runtime 404s reach the Worker. The middleware
intercepts the 404 response from `app.render()` and converts it to a
301 redirect.

This works on `output: 'static'` even though Astro docs say
"middleware runs at build time only in static mode." The
`@astrojs/cloudflare` adapter still generates a Worker (because it
needs to handle `/api/*` SSR routes), and that Worker handles the
404 fallthrough — which IS where middleware runs.

---

## Environment gate — automatic on/off

The middleware uses **Astro's built-in `import.meta.env.PROD`** flag
to switch behaviour automatically based on environment:

```ts
const FALLBACK_REDIRECT_ENABLED = import.meta.env.PROD;
```

| Command / environment | `import.meta.env.PROD` | Fallback |
|---|---|---|
| `astro dev` (localhost) | `false` | OFF — real 404s render `404.astro` |
| `astro build` | `true` | ON — included in production output |
| Cloudflare Pages (any deploy — preview or production) | `true` | ON — fallback fires |
| `astro preview` (local prod simulation) | `true` | ON — note: same as Cloudflare |

**Why automatic instead of a manual toggle:**

- Zero risk of forgetting to flip at launch
- Zero risk of accidentally shipping it disabled
- Localhost always shows real 404s so missing routes, broken
  internal links, and typos surface during development
- One mental model: localhost = dev, anywhere else = live

**Note:** `astro preview` activates the fallback because it's a
production build. If you need real 404s locally for QA, use
`astro dev`, not `astro preview`.

## Implementation

### File: `src/middleware.ts`

```ts
/**
 * Rank Math "Fallback Behaviour → Redirect to Homepage" equivalent.
 * Any unmatched URL → 301 to "/" instead of rendering the 404 page.
 *
 * Runs inside the @astrojs/cloudflare Worker. Triggers ONLY on the
 * 404 fallthrough — valid pages, static assets, and SSR routes are
 * unaffected.
 */
import { defineMiddleware } from 'astro:middleware';

// Environment gate — false on localhost (`astro dev`), true on Cloudflare.
const FALLBACK_REDIRECT_ENABLED = import.meta.env.PROD;

const NEVER_FALLBACK_PREFIXES: readonly string[] = [
  '/_astro/',          // Astro build assets (content-hashed)
  '/_image',           // Astro image service
  '/_server-islands/', // Astro server islands
  '/__astro_',         // Astro Cloudflare adapter prerender endpoints
  '/api/',             // SSR API routes
  '/images/',          // Static images
  '/videos/',          // Static videos
  '/fonts/',           // Self-hosted fonts
  '/icons/',           // Static icons
];

const NEVER_FALLBACK_PATHS: ReadonlySet<string> = new Set([
  '/robots.txt',
  '/sitemap.xml',
  '/sitemap-0.xml',
  '/sitemap-index.xml',
  '/favicon.ico',
  '/favicon.svg',
  '/manifest.webmanifest',
  '/site.webmanifest',
]);

const SAFE_METHODS: ReadonlySet<string> = new Set(['GET', 'HEAD']);

export const onRequest = defineMiddleware(async (context, next) => {
  const response = await next();
  if (!FALLBACK_REDIRECT_ENABLED) return response;
  if (response.status !== 404) return response;

  const { pathname } = context.url;
  const method = context.request.method.toUpperCase();

  // Loop prevention.
  if (pathname === '/' || pathname === '') return response;

  // Only safe HTTP methods.
  if (!SAFE_METHODS.has(method)) return response;

  // Excluded specific files (crawler-facing files must stay 404).
  if (NEVER_FALLBACK_PATHS.has(pathname)) return response;

  // Excluded path prefixes (assets, APIs, internal endpoints).
  for (const prefix of NEVER_FALLBACK_PREFIXES) {
    if (pathname.startsWith(prefix)) return response;
  }

  // Broken hotlinks to non-HTML files should 404, not redirect.
  const lastSegment = pathname.split('/').pop() ?? '';
  if (lastSegment.includes('.') && !lastSegment.endsWith('.html')) {
    return response;
  }

  return new Response(null, {
    status: 301,
    headers: {
      'Location': '/',
      'Cache-Control': 'public, max-age=300',
      'X-Redirect-Reason': 'fallback-to-home',
    },
  });
});
```

### File: `public/_redirects`

Reserve `_redirects` for **specific** old → new mappings. Do NOT add
a `/* / 301` wildcard here — it will break the build.

```
# ── WordPress slug migrations ──
/old-service-page                    /services/joint-injections/   301
/wordpress-style-permalink/2021/*    /blog/:splat                  301

# ── Slug-mismatch redirects ──
/privacy-notice                  /privacy-policy/   301

# ── Rank Math fallback ─ implemented in src/middleware.ts ─────────
# Do NOT add a wildcard rule here — the build will fail.
```

### File: `astro.config.mjs` (no changes required)

The middleware works with `output: 'static'` (Astro 6 default). No
config changes needed. Optional: add specific known-old → known-new
redirects in the `redirects:` block for build-time static redirects:

```js
export default defineConfig({
  // ...existing config...
  redirects: {
    '/old-service-page':   '/services/joint-injections/',
    '/2021/blog/welcome':  '/blog/welcome/',
    '/contact-us':         '/contact/',
  },
});
```

### File: `src/pages/404.astro` (unchanged)

Keep your designed 404 page. The middleware intercepts BEFORE
`404.astro` renders, but the page remains as defence-in-depth — if
middleware is ever disabled, it falls back to the branded 404.

## Verification

### Local build

```bash
npm run build
```

Must complete without errors. The middleware runs at build
compile-time validation; the actual 404 → 301 logic only fires at
runtime on Cloudflare.

### Production (after deploy to Cloudflare Pages preview/main)

```bash
SITE=https://your-site.pages.dev

# 1. Valid route — should return 200
curl -sI $SITE/about-us/ | head -3

# 2. Specific redirect — should return 301 to mapped URL
curl -sI $SITE/old-service-page | head -5

# 3. Random unmatched URL — should return 301 to /
curl -sI $SITE/totally-random-url | head -5

# 4. Missing image — should remain 404
curl -sI $SITE/images/does-not-exist.webp | head -3

# 5. Missing API path — should remain 404
curl -sI $SITE/api/does-not-exist | head -3

# 6. Homepage — should serve 200
curl -sI $SITE/ | head -3

# 7. Inspect the redirect reason header (debug)
curl -sI $SITE/anything-not-a-real-page | grep -i 'x-redirect-reason'
```

Expected:

| # | Path | Status | Location | Notes |
|---|---|---|---|---|
| 1 | `/about-us/` | `200` | — | Static asset |
| 2 | `/old-service-page` | `301` | `/services/joint-injections/` | _redirects rule |
| 3 | `/totally-random-url` | `301` | `/` | Middleware fallback |
| 4 | `/images/does-not-exist.webp` | `404` | — | NEVER_FALLBACK_PREFIXES |
| 5 | `/api/does-not-exist` | `404` | — | NEVER_FALLBACK_PREFIXES |
| 6 | `/` | `200` | — | Loop prevention |
| 7 | (any random) | — | — | Should output `X-Redirect-Reason: fallback-to-home` |

## Edge cases handled

| Case | Behaviour | Why |
|---|---|---|
| Valid page (`/about/`) | 200 | Static asset priority — middleware doesn't run |
| Specific old → new redirect | 301 to mapped URL | `_redirects` runs before Worker |
| Random unmatched URL | 301 to `/` | Middleware catches 404 |
| Missing image | 404 | `NEVER_FALLBACK_PREFIXES` excludes `/images/` |
| Missing API path | 404 | `NEVER_FALLBACK_PREFIXES` excludes `/api/` |
| Missing font/video/icon | 404 | Same — asset paths excluded |
| `/robots.txt` missing | 404 | `NEVER_FALLBACK_PATHS` |
| `/sitemap.xml` missing | 404 | Same |
| Missing `/favicon.ico` | 404 | Same |
| Homepage `/` | 200 | Static priority + explicit loop prevention |
| POST to random URL | 404 (preserved) | `SAFE_METHODS` excludes POST |
| `/missing.pdf` (file with extension) | 404 | Extension check excludes non-`.html` files |
| URL with query string | 301 to `/` (query stripped) | Same as Rank Math |
| Trailing slash mismatch | Astro normalises before middleware | Per `trailingSlash` config |

---

## What this does NOT do

- Does NOT preserve query strings or fragments on the fallback redirect.
- Does NOT redirect `POST` data to homepage with the body intact.
- Does NOT inspect response body or content. Pure URL-pattern matching.
- Does NOT replace the need for proper specific old → new redirects
  for SEO-valuable URLs.

## Per-clinic deployment checklist

When applying this pattern to a new clinic site cloned from the base
template:

**During development (localhost — fallback automatically OFF):**

- [ ] `src/middleware.ts` exists (copy from base template).
- [ ] `FALLBACK_REDIRECT_ENABLED = import.meta.env.PROD` (default).
- [ ] `public/_redirects` does NOT contain a `/* / 301` wildcard rule.
- [ ] `public/_redirects` may contain specific old → new redirects.
- [ ] If the project adds custom asset paths beyond
      `/images/`, `/videos/`, `/fonts/`, `/icons/`, `/_astro/`, the
      new prefix is added to `NEVER_FALLBACK_PREFIXES` in middleware.
- [ ] `npm run build` completes successfully.
- [ ] On `astro dev`, real 404s render the branded 404.astro page
      (verify by visiting any non-existent URL on localhost).

**At go-live (no code change required — fallback is already on for
any Cloudflare deploy):**

- [ ] Cloudflare Pages preview deploy passes the 7-curl verification
      above (random URLs 301 to `/`).
- [ ] Search Console submitted with new sitemap.
- [ ] Server logs / Cloudflare Analytics monitored for 14 days
      post-launch — high-frequency hits to `X-Redirect-Reason:
      fallback-to-home` for the same URL means a missing specific
      redirect that should be added.

**To temporarily disable the fallback on a Cloudflare deploy** (e.g.
debugging a specific 404), change the gate to `false`:

```ts
const FALLBACK_REDIRECT_ENABLED = false;
```

Remember to revert before merging to main.

---

## Reverting

To disable the fallback (e.g. you decide to let real 404s be 404s):

1. Delete `src/middleware.ts`.
2. Run `npm run build`.

The branded `src/pages/404.astro` will then render normally for
unmatched URLs.
