---
name: Primary colour usage rules
description: Rules for using the project's primary colour intentionally and consistently, especially on legal/policy pages and sidebar navigation
type: feedback
---

Always use the project's primary colour intentionally and consistently across the website.

**Why:** User added this as a design standard after noticing interface elements defaulting to dull grey where the brand colour should have been applied — particularly in sidebar navs, legal page section numbers, and active states.

**How to apply:** Before finalising any page, check that key accents, active states, labels, and section markers use the primary colour where appropriate. Never leave brand-connecting UI elements in flat grey.

Required usage:
- Primary colour for important accents, active states, key highlights, and interactive elements
- On legal/policy/sidebar pages: active item, progress indicators, section markers, key labels, important UI accents
- Usage must be balanced, premium, and consistent — not random

Rules:
- Do not leave important interface elements in dull grey if they should visually connect to the brand
- Do not use primary colour in a way that harms readability
- Maintain accessible contrast at all times
- Use lighter/darker shades for hover, active, muted, or background states

For legal pages, policy pages, sticky side navigation, T&Cs, cookie policy, privacy policy:
- Active section states must clearly use the primary colour
- Scroll indicators or progress accents can use the primary colour
- Highlighted numbers, labels, icons, and small accent elements should align with the primary colour system
- The page must still feel clean, premium, and readable
